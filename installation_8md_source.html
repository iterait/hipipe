4 Requirements
5 ------------
6 ---
7 
8 Officially supported systems are Ubuntu 18.04+ and Arch Linux, although __hipipe__ should
9 work on any recent enough system. The __hipipe core__ is a pure C++ library that by default
10 depends on [Boost C++ Libraries](http://www.boost.org/) (v1.61+ with Boost::Python
11 is required) and [OpenCV](http://opencv.org/) for image conversion between C++ and Python.
12 Python bindings and OpenCV support can be disabled, see Advanced Build Options section below.
13 
14 If you plan to use the full functionality (this is the default behavior),
15 install all the requirements by one of the following commands:
16 
17 ```
18 # Arch Linux
19 pacman -S git base-devel cmake boost opencv python python-numpy
20 
21 # Ubuntu 18.04+
22 apt install git build-essential cmake libboost-all-dev libopencv-dev python3-dev python3-numpy
23 ```
24 
25 If you plan to use [TensorFlow C++ API](https://www.tensorflow.org/api_guides/cc/guide),
26 please install the TensorFlow C++ library into your system using
27 [tensorflow_cc](https://github.com/FloopCZ/tensorflow_cc) project. This is not neccessary
28 if you only want to use TensorFlow in Python and it is disabled by default.

42 Build & Install
43 ---------------
44 ---
45 
46 Use the following for system-wide installation:
47 
48 ```
49 mkdir build && cd build
50 cmake ..
51 make -j4
52 ctest --output-on-failure
53 sudo make install
54 ```
55 
56 Or use the following for userspace installation:
57 
58 ```
59 mkdir build && cd build
60 cmake -DCMAKE_INSTALL_PREFIX=~/.local ..
61 make -j4
62 ctest --output-on-failure
63 make install
64 ```
65 
66 For userspace installation, don't forget to set the appropriate
67 environmental variables, e.g., add the following to your `.bashrc` / `.zshrc`:
68 ```
69 # register ~/.local system hierarchy
70 export PATH="${HOME}/.local/bin:$PATH"
71 export LIBRARY_PATH="${HOME}/.local/lib:$LIBRARY_PATH"
72 export LD_LIBRARY_PATH="${HOME}/.local/lib:$LD_LIBRARY_PATH"
73 export CPLUS_INCLUDE_PATH="${HOME}/.local/include:$CPLUS_INCLUDE_PATH"
74 ```

76 Advanced Build Options
77 ----------------------
78 ---
79 
80 There are multiple build options that can be set when configuring the project
81 with CMake. For instance, if you don't want to build tests because it takes a
82 lot of time, and you are also not interested in Python interoperability,
83 you may use the `BUILD_TEST` and `BUILD_PYTHON` flags as follows:
84 
85 ```
86 cmake -DHIPIPE_BUILD_TEST=OFF -DHIPIPE_BUILD_PYTHON=OFF ..
87 ```
88 
89 The full list of supported options is the following:
90 
91 | Option | Description | Default |
92 |----------------------------|-------------------------------------------------------------------------------|--------------|
93 | HIPIPE_BUILD_TEST | Build tests. | ON |
94 | HIPIPE_BUILD_DOC | Build documentation. | OFF |
95 | HIPIPE_BUILD_PYTHON | Build Python functionality. | ON |
96 | HIPIPE_BUILD_PYTHON_OPENCV | Build Python OpenCV converters (requires HIPIPE_BUILD_PYTHON). | ON |
97 | HIPIPE_BUILD_TENSORFLOW | Build TensorFlow functionality (unnecessary if you use TensorFlow in Python). | OFF |
98 | HIPIPE_BUILTIN_RANGEV3 | Install and use the built-in Range-v3 library. | ON |
99 | CMAKE_INSTALL_PREFIX | The path where hipipe will be installed. | OS-dependent |
100 | CMAKE_CXX_COMPILER | The compiler command to be used, e.g., g++ or clang++. | OS-dependent |

102 Development
103 -----------
104 ---
105 
106 Are you missing a feature or have you found a bug? You can either fill an issue on our
107 GitHub (https://github.com/iterait/hipipe) or even better, become a developer!
108 
109 There are no special requirements for development, just a few hints:
110 - Header files are under `include/` directory.
111 - Source files are under `src/` directory.
112 - Unit tests are under `test/` directory.
113 - Any new functionality has to be accompanied by unit tests.
114 - If you add a new header, don't forget to add it to the corresponding accumulating header in parent folder.
115  For instance, `include/core/stream.hpp` lists all the headers under `include/core/stream` directory.
116 - If you add a new test, don't forget to register it in `CMakeLists.txt` located in the same folder as the test.
117 - To build and run a single unit test, run e.g.,:
118 ```
119 make test.core.stream.drop
120 ctest --output-on-failure -R test.core.stream.drop
121 ```
122 - To build and run a single test that requires building a Python module, run e.g.,:
123 ```
124 make test.core.python.stream.converter_py_cpp # note the py_cpp suffix
125 ctest --output-on-failure -R test.core.python.stream.converter
126 ```
