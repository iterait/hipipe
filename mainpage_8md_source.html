6 Preface
7 -------
8 ---
9 
10 __HiPipe__ is a C++ library for efficient data processing. Its main purpose is to simplify
11 and acclelerate data preparation for deep learning models, but it is generic enough to be used
12 in many other areas.
13 
14 __HiPipe__ lets the programmer build intuitive data streams that transform,
15 combine and filter the data that pass through. Those streams are compiled,
16 batched, and asynchronous, therefore maximizing the utilization of the provided
17 hardware.
18 
19 The complete source code can be downloaded from our official GitHub
20 [repository](https://github.com/iterait/hipipe).
21 
22 **This project is under heavy development. The API is continuously changing without regard
23 to backward compatibility.**

25 Quick Start
26 -----------
27 ---
28 
29 Before proceeding, please install __HiPipe__ by following the
30 \ref installation "installation guide".
31 
32 __HiPipe__ provides many smaller utilities, such as loading \ref CSV "CSV files" to
33 \ref Dataframe "dataframes", \ref Base64 "base64 decoders",
34 \ref Groups "data splitting" and [much more](modules.html). However, probably the most
35 useful funcionality of the library is \ref Stream "data stream processing".
36 
37 Before we dive into details, we recommend to get familiar with the concept of C++
38 ranges and [Range-v3](http://ericniebler.github.io/range-v3/) library by Eric Niebler.
39 Ranges will soon become a fundamental part of C++ standard and Range-v3 library
40 is an experimental implementation of the proposal. To sum up the idea, a range can be
41 thought of as a pair of begin/end iterators with potentially infinite distance.
42 
43 Now, a _stream_ in the context of __HiPipe__ is a range of _batches_, where each batch
44 contains one or more _columns_. To demonstrate this concept on an example,
45 let us assume that we have a table of four users and each user has
46 properties: `login` and `age`:
47 
48 | login | age |
49 |-------|-----|
50 | marry | 24 |
51 | ted | 41 |
52 | anna | 16 |
53 | josh | 59 |
54 
55 A stream made of such data with batch size of two may look as following:
56 
57 ![diagram of stream, batch and column](./resources/stream_batch_column.png)
58 
59 Of course, the stream may as well be loaded from a file or a database and may even
60 be infinite. __HiPipe__ provides \ref Stream "many tools" to manipulate such
61 streams using the pipe "|" operator. The following example demonstrates the main idea of
62 stream pipelining (see the full example in [example.cpp](example_8cpp_source.html)):
63 
64 ```{.cpp}
65 namespace hps = hipipe::stream;
66 using hps::from; using hps::to; using hps::by; using hps::dim;
67 
68 HIPIPE_DEFINE_COLUMN(login, std::string) // helper macro to define a column of strings
69 HIPIPE_DEFINE_COLUMN(age, int)
70 
71 std::vector<std::string> logins = {"marry", "ted", "anna", "josh"};
72 std::vector<int> ages = { 24, 41, 16, 59};
73 
74 auto stream = ranges::view::zip(logins, ages)
75 
76  // create a batched stream out of the raw data
77  | hps::create<login, age>(2)
78 
79  // make everyone older by one year
80  | hps::transform(from<age>, to<age>, [](int a) { return a + 1; })
81 
82  // increase each letter in the logins by one (i.e., a->b, e->f ...)
83  | hps::transform(from<login>, to<login>, [](char c) { return c + 1; }, dim<2>)
84 
85  // increase the ages by the length of the login
86  | hps::transform(from<login, age>, to<age>, [](std::string l, int a) {
87  return a + l.length();
88  })
89 
90  // probabilistically rename 50% of the people to "buzz"
91  | hps::transform(from<login>, to<login>, 0.5, [](std::string) -> std::string {
92  return "buzz";
93  })
94 
95  // drop the login column from the stream
96  | hps::drop<login>
97 
98  // introduce the login column back to the stream
99  | hps::transform(from<age>, to<login>, [](int a) {
100  return "person_" + std::to_string(a) + "_years_old";
101  })
102 
103  // filter only people older than 30 years
104  | hps::filter(from<login, age>, by<age>, [](int a) { return a > 30; })
105 
106  // asynchronously buffer the stream during iteration
107  | hps::buffer(2);
108 
109 // extract the ages from the stream to std::vector
110 ages = hps::unpack(stream, from<age>);
111 assert((ages == std::vector<int>{45, 64}));
112 ```
113 
114 For more information, please refer to the [modules](modules.html) documentation section
115 and browse the API reference.

117 Multidimensional data
118 ---------------------
119 ---
120 
121 In __HiPipe__, when we talk about multidimensional data, we are talking about nested
122 ranges. For instance, `std::vector<int>` is a one dimensional vector and
123 `std::vector<std::vector<int>>` is a two dimensional vector.
124 
125 Most of the stream modifiers have a special parameter called `dim<N>`,
126 that denotes in which dimension should the modifier operate. For instance,
127 consider the transform() from our previous example:
128 
129 ```{.cpp}
130 HIPIPE_DEFINE_COLUMN(login, std::string)
131 
132 ...
133 
134  // This transformation is applied in dimension 1 (default), so in the case of the
135  // login column, the transformation function expects std::string.
136  | hps::transform(from<login>, to<login>, [](std::string l) { return l + "_abc"; })
137 
138  // This transformation is applied in dimension 2, which for std::string means
139  // that it is applied on each letter.
140  | hps::transform(from<login>, to<login>, dim<2>, [](char c) { return c + 1; })
141 
142  // And this transformation is applied in dimension 0, which denotes that the function
143  // expects the whole batch.
144  | hps::transform(from<login>, to<login>, dim<0>, [](std::vector<std::string> login_batch) {
145  login_batch.push_back("new_login"); // this inserts a new login to the current batch
146  return login_batch;
147  })
148 ```
149 
150 In __HiPipe__, there are no constraints on the shape of the columns. Batch
151 sizes can differ batch to batch, as well as the sizes of the data across a
152 single dimension. For instance, it is possible to have a batch of variably long
153 sequences (e.g., a batch of video frames from videos of different length) and
154 stream modifiers, such as transform() and filter(), will handle it in any dimension
155 you choose. To define a column containing multidimensional data, the most simple
156 solution is to wrap the data in `std::vector` (but any reasonable container will
157 suffice):
158 
159 ```
160 HIPIPE_DEFINE_COLUMN(stock_price_evolution, std::vector<double>)
161 ```
